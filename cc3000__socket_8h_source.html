222  * Name: cc3000_accept_impl
223  *
224  * Decription:
225  * accept a connection on a socket:
226  * This function is used with connection-based socket types
227  * (SOCK_STREAM). It extracts the first connection request on the
228  * queue of pending connections, creates a new connected socket, and
229  * returns a new file descriptor referring to that socket.
230  * The newly created socket is not in the listening state.
231  * The original socket sd is unaffected by this call.
232  * The argument sd is a socket that has been created with socket(),
233  * bound to a local address with bind(), and is listening for
234  * connections after a listen(). The argument addr is a pointer
235  * to a sockaddr structure. This structure is filled in with the
236  * address of the peer socket, as known to the communications layer.
237  * The exact format of the address returned addr is determined by the
238  * socket's address family. The addrlen argument is a value-result
239  * argument: it should initially contain the size of the structure
240  * pointed to by addr, on return it will contain the actual
241  * length (in bytes) of the address returned.
242  *
243  * Input Parameters:
244  * sd socket descriptor (handle)
245  * addr the argument addr is a pointer to a sockaddr structure
246  * This structure is filled in with the address of the
247  * peer socket, as known to the communications layer.
248  * determined. The exact format of the address returned
249  * addr is by the socket's address sockaddr.
250  * On this version only AF_INET is supported.
251  * This argument returns in network order.
252  * addrlen The addrlen argument is a value-result argument:
253  * it should initially contain the size of the structure
254  * pointed to by addr.
255  *
256  * Returned Value:
257  * For socket in blocking mode:
258  * On success, socket handle. on failure negative
259  * For socket in non-blocking mode:
260  * - On connection establishment, socket handle
261  * - On connection pending, SOC_IN_PROGRESS (-2)
262  * - On failure, SOC_ERROR (-1)
263  *

351 /****************************************************************************
352  * Name: cc3000_select_impl
353  *
354  * Decription:
355  * Monitor socket activity
356  * Select allow a program to monitor multiple file descriptors,
357  * waiting until one or more of the file descriptors become
358  * "ready" for some class of I/O operation
359  *
360  * NOTE: If the timeout value set to less than 5ms it will automatically set
361  * to 5ms to prevent overload of the system
362  *
363  * Input Parameters:
364  * nfds the highest-numbered file descriptor in any of the
365  * three sets, plus 1.
366  * readfds socket descriptors list for read monitoring
367  * writefds socket descriptors list for write monitoring
368  * exceptfds socket descriptors list for exception monitoring
369  * timeout is an upper bound on the amount of time elapsed
370  * before select() returns. Null means infinity
371  * timeout. The minimum timeout is 5 milliseconds,
372  * less than 5 milliseconds will be set
373  * automatically to 5 milliseconds.
374  *
375  * Returned Value:
376  * On success, select() returns the number of file descriptors
377  * contained in the three returned descriptor sets (that is, the
378  * total number of bits that are set in readfds, writefds,
379  * exceptfds) which may be zero if the timeout expires before
380  * anything interesting happens.
381  * On error, -1 is returned.
382  * *readfds - return the sockets on which Read request will
383  * return without delay with valid data.
384  * *writefds - return the sockets on which Write request
385  * will return without delay.
386  * *exceptfds - return the sockets which closed recently.
387  *
388  ****************************************************************************/

419  * parameter, initially containing the size of the buffer

422  * value is to be supplied or returned, option_value may be NULL.

425  * The only protocol level supported in this version

427  * 1. SOCKOPT_RECV_TIMEOUT (optname)
