40 /* For the purposes of this driver, an Audio device is any device that
41  * generates, records, mixes, or otherwise modifies audio data in any format,
42  * such as PCM, MP3, AAC, etc.
43  *
44  * The Audio driver is split into two parts:
45  *
46  * 1) An "upper half", generic driver that provides the comman Audio interface
47  * to application level code, and
48  * 2) A "lower half", platform-specific driver that implements the low-level
49  * controls to configure and communicate with the audio device(s).
50  */

76 /* IOCTL Commands ***********************************************************/
77 /* The Audio module uses a standard character driver framework. However, a
78  * lot of the Audio driver functionality is configured via a device control
79  * interface, such as sampling rate, volume, data format, etc.
80  * The Audio ioctl commands are listed below:
81  *
82  * AUDIOIOC_GETCAPS - Get the Audio Device Capabilities
83  *
84  * ioctl argument: Pointer to the audio_caps_s structure to receive the
85  * capabilities info. The "len" and "type" fields should
86  * be filled in prior to calling this ioctl. To get
87  * overall capabilities, specify the type as
88  * AUDIO_TYPE_QUERY, otherwise specify any type that was
89  * reported by the device during the QUERY.
90  *
91  * AUDIOIOC_CONFIGURE - Configure device for the specified mode
92  *
93  * ioctl argument: Pointer to the audio_caps_s structure which identifies
94  * the capabilities to configure for.
95  *
96  * AUDIOIOC_SHUTDOWN - Shutdown the device.
97  *
98  * ioctl argument: None
99  *
100  * AUDIOIOC_START - Start Audio streaming
101  *
102  * ioctl argument: None
103  *
104  * AUDIOIOC_STOP - Stop Audio streaming
105  *
106  * ioctl argument: None
107  */
