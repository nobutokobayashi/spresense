72 /****************************************************************************
73  * Name: up_tls_info
74  *
75  * Description:
76  * Return the TLS information structure for the currently executing thread.
77  * When TLS is enabled, up_createstack() will align allocated stacks to
78  * the TLS_STACK_ALIGN value. An instance of the following structure will
79  * be implicitly positioned at the "lower" end of the stack. Assuming a
80  * "push down" stack, this is at the "far" end of the stack (and can be
81  * clobbered if the stack overflows).
82  *
83  * If an MCU has a "push up" then that TLS structure will lie at the top
84  * of the stack and stack allocation and initialization logic must take
85  * care to preserve this structure content.
86  *
87  * The stack memory is fully accessible to user mode threads.
88  *
89  * Input Parameters:
90  * None
91  *
92  * Returned Value:
93  * A pointer to TLS info structure at the beginning of the STACK memory
94  * allocation. This is essentially an application of the TLS_INFO(sp)
95  * macro and has a platform dependency only in the manner in which the
96  * stack pointer (sp) is obtained and interpreted.
97  *
98  ****************************************************************************/
99 
100 static inline FAR struct tls_info_s *up_tls_info(void)
101 {
102  DEBUGASSERT(!up_interrupt_context());
103  return TLS_INFO((uintptr_t)up_getsp());
104 }
