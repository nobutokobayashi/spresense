51 /* The Global Interrupt Controller (GIC) collects up to 128 interrupt
52  * requests from all i.MX 6Dual/6Quad sources and provides an interface to
53  * each of the CPU cores.
54  *
55  * The first 32 interrupts are used for interrupts that are private to the
56  * CPUs interface. interrupts besides the private CPU are also hooked up to
57  * the GIC in the same order.
58  *
59  * Each interrupt can be configured as a normal or a secure interrupt.
60  * Software force registers and software priority masking are also
61  * supported. The following table describes the ARM interrupt sources.
62  */
63 
64 /* Private Peripheral Interrupts (PPI) **************************************/
65 /* Each Cortex-A9 processor has private interrupts, ID0-ID15, that can only
66  * be triggered by software. These interrupts are aliased so that there is
67  * no requirement for a requesting Cortex-A9 processor to determine its own
68  * CPU ID when it deals with SGIs. The priority of an SGI depends on the
69  * value set by the receiving Cortex-A9 processor in the banked SGI priority
70  * registers, not the priority set by the sending Cortex-A9 processor.
71  */
72
