60 /* The following flags may be set in the set of flags by the lower, device-
61  * interfacing layer before calling through the socket layer callback. The
62  * TCP_ACKDATA, XYZ_NEWDATA, and TCP_CLOSE flags may be set at the same time,
63  * whereas the others are mutually exclusive.
64  *
65  * Connection Specific Events: These are events that may be notified
66  * through callback lists residing in TCP, UDP, or PKT port connection
67  * structures:
68  *
69  * TCP_ACKDATA IN: Signifies that the outstanding data was ACKed and
70  * the socket layer should send out new data instead
71  * of retransmitting the last data (TCP only)
72  * OUT: Input state must be preserved on output.
73  *
74  * TCP_NEWDATA IN: Set to indicate that the peer has sent us new data.
75  * UDP_NEWDATA OUT: Cleared (only) by the socket layer logic to indicate
76  * PKT_NEWDATA that the new data was consumed, suppressing further
77  * IEEE802154_NEWDATA attempts to process the new data.
78  *
79  * TCP_SNDACK IN: Not used; always zero
80  * OUT: Set by the socket layer if the new data was consumed
81  * and an ACK should be sent in the response. (TCP only)
82  *
83  * TCP_REXMIT IN: Tells the socket layer to retransmit the data that
84  * was last sent. (TCP only)
85  * OUT: Not used
86  *
87  * TCP_POLL IN: Used for polling the socket layer. This is provided
88  * UDP_POLL periodically from the drivers to support (1) timed
89  * PKT_POLL operations, and (2) to check if the socket layer has
90  * IEEE802154_POLL data that it wants to send. These are socket oriented
91  * callbacks where the context depends on the specific
92  * set
93  * OUT: Not used
94  *
95  * TCP_BACKLOG IN: There is a new connection in the backlog list set
96  * up by the listen() command. (TCP only)
97  * OUT: Not used
98  *
99  * TCP_CLOSE IN: The remote host has closed the connection, thus the
100  * connection has gone away. (TCP only)
101  * OUT: The socket layer signals that it wants to close the
102  * connection. (TCP only)
103  *
104  * TCP_ABORT IN: The remote host has aborted the connection, thus the
105  * connection has gone away. (TCP only)
106  * OUT: The socket layer signals that it wants to abort the
107  * connection. (TCP only)
108  *
109  * TCP_CONNECTED IN: We have got a connection from a remote host and have
110  * set up a new connection for it, or an active connection
111  * has been successfully established. (TCP only)
112  * OUT: Not used
113  *
114  * TCP_TIMEDOUT IN: The connection has been aborted due to too many
115  * retransmissions. (TCP only)
116  * OUT: Not used
117  *
118  * Device Specific Events: These are events that may be notified through
119  * callback lists residing in the network device structure.
120  *
121  * ICMP_NEWDATA IN: Set to indicate that the peer has sent us new data.
122  * ICMPv6_NEWDATA OUT: Cleared (only) by the socket layer logic to indicate
123  * that the new data was consumed, suppressing further
124  * attempts to process the new data.
125  *
126  * ARP_POLL IN: Used for polling the socket layer. This is provided
127  * periodically from the drivers to support (1) timed
128  * operations, and (2) to check if the ARP layer needs
129  * to send an ARP request. This is a device oriented
130  * event, not associated with a socket.
131  * OUT: Not used
132  *
133  * ICMP_POLL IN: Used for polling the socket layer. This is provided
134  * periodically from the drivers to support (1) timed
135  * operations, and (2) to check if the ICMP layer needs
136  * to send an ARP request. This is a device oriented
137  * event, not associated with a socket. This differs
138  * from ICMPv6_POLL only in that the appdata pointer
139  * is set differently
140  * OUT: Not used
141  *
142  * ICMPv6_POLL IN: Used for polling the socket layer. This is provided
143  * periodically from the drivers to support (1) timed
144  * operations, and (2) to check if the ICMP layer needs
145  * to send an ARP request. This is a device oriented
146  * event, not associated with a socket. This differs
147  * from ICMP_POLL only in that the appdata pointer
148  * is set differently
149  * OUT: Not used
150  *
151  * IPFWD_POLL IN: Used for polling for forwarded packets layer. This
152  * is provided periodically from the drivers to support
153  * to check if there is a packet waiting to be forward
154  * on the device. This is a device oriented event,
155  * not associated with a socket. The appdata pointer
156  * The appdata pointer is not used in this case.
157  * OUT: Not used
158  *
159  * ICMP_ECHOREPLY IN: An ICMP Echo Reply has been received. Used to support
160  * ICMP ping from the socket layer. (ICMPv4 only)
161  * OUT: Cleared (only) by the socket layer logic to indicate
162  * that the reply was processed, suppressing further
163  * attempts to process the reply.
164  *
165  * ICMPv6_ECHOREPLY IN: An ICMP Echo Reply has been received. Used to support
166  * ICMP ping from the socket layer. (ICMPv6 only)
167  * OUT: Cleared (only) by the socket layer logic to indicate
168  * that the reply was processed, suppressing further
169  * attempts to process the reply.
170  *
171  * NETDEV_DOWN: IN: The network device has been taken down.
172  * OUT: Not used
173  */

247  * nxtconn - Supports a singly linked list that supports connection

448 /****************************************************************************
449  * Send data on the current connection.
450  *
451  * This function is used to send out a single segment of TCP data. Only
452  * socket logic that have been invoked by the lower level for event
453  * processing can send data.
454  *
455  * The amount of data that actually is sent out after a call to this
456  * function is determined by the maximum amount of data TCP allows. The
457  * network will automatically crop the data so that only the appropriate
458  * amount of data is sent. The mss field of the TCP connection structure
459  * can be used to determine the amount of data that actually will be sent.
460  *
461  * Note: This function does not guarantee that the sent data will
462  * arrive at the destination. If the data is lost in the network, the
463  * TCP socket layer will be invoked with the TCP_REXMIT flag set. The
464  * socket layer will then have to resend the data using this function.
465  *
466  * data A pointer to the data which is to be sent.
467  *
468  * len The maximum amount of data bytes to be sent.
469  *
470  ****************************************************************************/
471 
472 void devif_send(FAR struct net_driver_s *dev, FAR const void *buf, int len);
