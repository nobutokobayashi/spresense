548  * Determine the address of the MMU page table. Regardless of the memory
549  * configuration, we will keep the page table in the SAMA5's internal SRAM.
550  * We will always attempt to use the bottom 16KB of internal SRAM for the
551  * page table, but there are a few conditions that affect this:
552  *
553  * 1) If CONFIG_ARCH_ROMPGTABLE, then the page table resides in ROM and we
554  * will not use any page table in RAM.

699 /* Level 2 Page table start addresses.
700  *
701  * 16Kb of memory is reserved hold the page table for the virtual mappings. A
702  * portion of this table is not accessible in the virtual address space (for
703  * normal operation). There is this large whole in the physcal address space
704  * for which there will never be level 1 mappings:
705  *
706  * 0x80000000-0xefffffff: Undefined (1.75 GB)
707  *
708  * That is the offset where the main L2 page tables will be positioned. This
709  * corresponds to page table offsets 0x000002000 up to 0x000003c00. That
710  * is 1792 entries, each mapping 4KB of address for a total of 7MB of virtual
711  * address space)
712  *
713  * Up to two L2 page tables may be used:
714  *
715  * 1) One mapping the vector table. However, L2 page tables must be aligned
716  * to 1KB address boundaries, so the minimum L2 page table size is then
717  * 1KB, mapping up a full megabyte of virtual address space.
718  *
719  * This L2 page table is only allocated if CONFIG_ARCH_LOWVECTORS is *not*
720  * defined. The SAMA5 boot-up logic will map the beginning of the boot
721  * memory to address 0x0000:0000 using both the MMU and the AXI matrix
722  * REMAP register. So no L2 page table is required.
723  *
724  * 2) If on-demand paging is supported (CONFIG_PAGING=y), than an additional
725  * L2 page table is needed. This page table will use the remainder of
726  * the address space.
727  */
