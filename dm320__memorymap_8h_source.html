54 /* DM320 Physical Memory Map, where:
55  *
56  * CW = cachable with write buffering
57  * -W = Write buffering only
58  * -- = Neither
59  *
60  * NOTE:
61  * 1. Most DM320 memory sections can be programmed to lie at different locations in
62  * the memory map. Therefore, much of the DM320 physical memory map is really
63  * board-specific and, as such, really belongs in the configs/<board>/include/board.h
64  * file rather than here.
65  *
66  * To handle all cases, this file defines a "default" physical memory map, but
67  * section address for most regions can be overriden if the same setting is
68  * defined in the board.h file (These defaults correspond to the product Neuros
69  * OSD memory configuration).
70  *
71  * 2. The DM320 only has a single control line for external peripherals. To support
72  * more than one peripheral, most hardware will use external memory decode logic,
73  * so that physical memory regions is in the board-specific files.
74  */
75
