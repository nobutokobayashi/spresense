193 /* Must be called periodically to clean up deallocations delayed by
194  * sched_kmm_free(). This may be done from either the IDLE thread or from a
195  * worker thread. The IDLE thread has very low priority and could starve
196  * the system for memory in some context.
197  */
198 
199 void sched_garbage_collection(void);
200 
201 /* Is is not a good idea for the IDLE threads to take the KMM semaphore.
202  * That can cause the IDLE thread to take processing time from higher
203  * priority tasks. The IDLE threads will only take the KMM semaphore if
204  * there is garbage to be collected.
205  *
206  * Certainly there is a race condition involved in sampling the garbage
207  * state. The looping nature of the IDLE loops should catch any missed
208  * garbage from the test on the next time arround.
209  */
210 
211 bool sched_have_garbage(void);
