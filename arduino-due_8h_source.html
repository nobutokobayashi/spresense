56 /* There are three user-controllable LEDs on board the Arduino Due board:
57  *
58  * LED GPIO
59  * ---------------- -----
60  * L Amber LED PB27
61  * TX Yellow LED PA21
62  * RX Yellow LED PC30
63  *
64  * LED L is connected to ground and can be illuminated by driving the PB27
65  * output high. The TX and RX LEDs are pulled high and can be illuminated by
66  * driving the corresponding
67  * GPIO output to low.
68  *
69  * These LEDs are not used by the board port unless CONFIG_ARCH_LEDS is
70  * defined. In that case, the usage by the board port is defined in
71  * include/board.h and src/sam_leds.c. The LEDs are used to encode OS-related
72  * events as follows:
73  *
74  * SYMBOL MEANING LED STATE
75  * L TX RX
76  * ------------------- ----------------------- -------- -------- --------
77  * LED_STARTED NuttX has been started OFF OFF OFF
78  * LED_HEAPALLOCATE Heap has been allocated OFF OFF OFF
79  * LED_IRQSENABLED Interrupts enabled OFF OFF OFF
80  * LED_STACKCREATED Idle stack created ON OFF OFF
81  * LED_INIRQ In an interrupt N/C GLOW OFF
82  * LED_SIGNAL In a signal handler N/C GLOW OFF
83  * LED_ASSERTION An assertion failed N/C GLOW OFF
84  * LED_PANIC The system has crashed N/C N/C Blinking
85  * LED_IDLE MCU is is sleep mode ------ Not used --------
86  *
87  * Thus if LED L is statically on, NuttX has successfully booted and is,
88  * apparently, running normmally. If LED RX is glowing, then NuttX is
89  * handling interrupts (and also signals and assertions). If TX is flashing
90  * at approximately 2Hz, then a fatal error has been detected and the system
91  */
92
