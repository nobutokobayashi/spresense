143 /****************************************************************************
144  * Name: pcf8574_lcd_backpack_register
145  *
146  * Description:
147  * Register a character driver that is a I2C LCD 'backpack' based on the
148  * PCF8574 I2C IO expander. It allows operation of the ever-popular HD44780
149  * based LCDs via I2C instead of parallel (saving a bunch of gpio lines).
150  *
151  * There are a multitude of these available from various sources (e.g. ebay).
152  * They typically vary by gpio-to-lcd pin mapping, and I2C addresss, but
153  * otherwise are functionally identical.
154  *
155  * The characters presented for codes depend on the masked rom of the
156  * particular LCD device attached, but the most common are:
157  * 'A00', which present 0x20-0x7f, and 0xa0-0xff', similar to JIS X 0201
158  * 'A02', which present 0x10-0xff, and include various european symbols
159  * In both cases, codes 0x00-0x07 map to the CGRAM characters, which can be
160  * loaded via ioctl SLCDIOC_CREATECHAR (q.v.).
161  *
162  * This driver supports the SLCD codec for various escape sequences; q.v.
163  * nuttx/lcd/slcd_codec.h for details. This driver supports the SLCD ioctl
164  * interface for various extended commands; q.v. nuttx/lcd/slcd_ioctl.h for
165  * details. This driver supports an additional ioctl for defining custom
166  * characters; see above for details.
167  *
168  * Parameters:
169  * devpath - path to device node; arbitrary, but typically '/dev/lcd0' or such
170  * i2c - the low-level i2c bus onto which to bind
171  * cfg - the board-specific configuration
172  *
173  ****************************************************************************/
174
