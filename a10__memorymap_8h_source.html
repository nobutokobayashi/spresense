499  * portion of this table is not accessible in the virtual address space (for
500  * normal operation). There are several large holes in the physical address
501  * space for which there will never be level 1 mappings:
502  *
503  * LI PAGE TABLE
504  * ADDRESS RANGE SIZE ENTRIES SECTIONS
505  * ----------------------- ------- -------------- ---------
506  * 0x0003:0000-0x01eb:ffff 275MB 0x0004-0x006c 26
507  * *(none usable) 0
508  * 0x01ec:0000-0x3fff:ffff 993MB 0x0078-0x0ffc 993
509  * *0x0400-0x0ffc 767
510  *
511  * And the largest is probably from the end of SDRAM through 0xfff0:0000.
512  * But the size of that region varies with the size of the installed SDRAM.
513  * It is at least:
514  *
515  * LI PAGE TABLE
516  * ADDRESS RANGE SIZE ENTRIES SECTIONS
517  * ----------------------- ------- -------------- ---------
518  * 0xc000:0000-0xffef:ffff 1022MB *0x3000-0x3ff8 1022

522  * * NOTE that the L2 page table entries must be aligned 1KB address
523  * boundaries.

529  * defined).
