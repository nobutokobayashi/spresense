123 /****************************************************************************
124  * Name: accept
125  *
126  * Decription:
127  * accept a connection on a socket:
128  * This function is used with connection-based socket types
129  * (SOCK_STREAM). It extracts the first connection request on the
130  * queue of pending connections, creates a new connected socket, and
131  * returns a new file descriptor referring to that socket.
132  * The newly created socket is not in the listening state.
133  * The original socket sd is unaffected by this call.
134  * The argument sd is a socket that has been created with socket(),
135  * bound to a local address with bind(), and is listening for
136  * connections after a listen(). The argument addr is a pointer
137  * to a sockaddr structure. This structure is filled in with the
138  * address of the peer socket, as known to the communications layer.
139  * The exact format of the address returned addr is determined by the
140  * socket's address family. The addrlen argument is a value-result
141  * argument: it should initially contain the size of the structure
142  * pointed to by addr, on return it will contain the actual
143  * length (in bytes) of the address returned.
144  *
145  * Input Parameters:
146  * sockfd socket descriptor (handle)
147  * addr the argument addr is a pointer to a sockaddr structure
148  * This structure is filled in with the address of the
149  * peer socket, as known to the communications layer.
150  * determined. The exact format of the address returned
151  * addr is by the socket's address sockaddr.
152  * On this version only AF_INET is supported.
153  * This argument returns in network order.
154  * addrlen The addrlen argument is a value-result argument:
155  * it should initially contain the size of the structure
156  * pointed to by addr.
157  *
158  * Returned Value:
159  * For socket in blocking mode:
160  * On success, socket handle. on failure negative
161  * For socket in non-blocking mode:
162  * - On connection establishment, socket handle
163  * - On connection pending, SOC_IN_PROGRESS (-2)
164  * - On failure, SOC_ERROR (-1)
165  *
166  ****************************************************************************/
167 
168 int accept(int sockfd, struct sockaddr *addr, socklen_t *addrlen);

221 /****************************************************************************
222  * Name: connect
223  *
224  * Decription:
225  * initiate a connection on a socket
226  * Function connects the socket referred to by the socket descriptor
227  * sd, to the address specified by addr. The addrlen argument
228  * specifies the size of addr. The format of the address in addr is
229  * determined by the address space of the socket. If it is of type
230  * SOCK_DGRAM, this call specifies the peer with which the socket is
231  * to be associated; this address is that to which datagrams are to be

233  * If the socket is of type SOCK_STREAM, this call attempts to make a

235  * address, which is an address in the communications space of the
236  * socket. Note that the function implements only blocking behavior
237  * thus the caller will be waiting either for the connection
238  * establishment or for the connection establishment failure.
239  *
240  * Input Parameters:
241  * sockfd socket descriptor (handle)
242  * addr specifies the destination addr. On this version
243  * only AF_INET is supported.
244  * addrlen contains the size of the structure pointed to by addr

253 /****************************************************************************
254  * Name: select
255  *
256  * Decription:
257  * Monitor socket activity
258  * Select allow a program to monitor multiple file descriptors,
259  * waiting until one or more of the file descriptors become
260  * "ready" for some class of I/O operation
261  *
262  * NOTE: If the timeout value set to less than 5ms it will automatically set
263  * to 5ms to prevent overload of the system
264  *
265  * Input Parameters:
266  * nfds the highest-numbered file descriptor in any of the
267  * three sets, plus 1.
268  * readfds socket descriptors list for read monitoring
269  * writefds socket descriptors list for write monitoring
270  * exceptfds socket descriptors list for exception monitoring
271  * timeout is an upper bound on the amount of time elapsed
272  * before select() returns. Null means infinity
273  * timeout. The minimum timeout is 5 milliseconds,
274  * less than 5 milliseconds will be set
275  * automatically to 5 milliseconds.
276  *
277  * Returned Value:
278  * On success, select() returns the number of file descriptors
279  * contained in the three returned descriptor sets (that is, the
280  * total number of bits that are set in readfds, writefds,
281  * exceptfds) which may be zero if the timeout expires before
282  * anything interesting happens.
283  * On error, -1 is returned.
284  * *readfds - return the sockets on which Read request will
285  * return without delay with valid data.
286  * *writefds - return the sockets on which Write request
287  * will return without delay.
288  * *exceptfds - return the sockets which closed recently.
289  *
290  ****************************************************************************/
291

298  * Name: setsockopt
299  *
300  * Decription:
301  * set socket options
302  * This function manipulate the options associated with a socket.
303  * Options may exist at multiple protocol levels; they are always
304  * present at the uppermost socket level.
305  * When manipulating socket options the level at which the option
306  * resides and the name of the option must be specified.
307  * To manipulate options at the socket level, level is specified as
308  * SOL_SOCKET. To manipulate options at any other level the protocol
309  * number of the appropriate protocol controlling the option is
310  * supplied. For example, to indicate that an option is to be
311  * interpreted by the TCP protocol, level should be set to the
312  * protocol number of TCP;
313  * The parameters value and value_len are used to access value -
314  * use for setsockopt(). For getsockopt() they identify a buffer
315  * in which the value for the requested option(s) are to
316  * be returned. For getsockopt(), value_len is a value-result
317  * parameter, initially containing the size of the buffer
318  * pointed to by option_value, and modified on return to
319  * indicate the actual size of the value returned. If no option
320  * value is to be supplied or returned, option_value may be NULL.
321  *
322  * NOTE: On this version the following two socket options are enabled:
323  * The only protocol level supported in this version
324  * is SOL_SOCKET (level).
325  * 1. SOCKOPT_RECV_TIMEOUT (option)
326  * SOCKOPT_RECV_TIMEOUT configures recv and recvfrom timeout
327  * in milliseconds.
328  * In that case value should be pointer to unsigned long.
329  * 2. SOCKOPT_NONBLOCK (option). sets the socket non-blocking mode on
330  * or off.
331  * In that case value should be SOCK_ON or SOCK_OFF (value).
332  *
333  * Input Parameters:
334  * sockfd socket handle
335  * level defines the protocol level for this option
336  * option defines the option name to Interrogate
337  * value specifies a value for the option
338  * value_len specifies the length of the option value
339  *
340  * Returned Value:
341  * On success, zero is returned. On error, -1 is returned
342  *
343  ****************************************************************************/
344 
345 int setsockopt(int sockfd, int level, int option, FAR const void *value, socklen_t value_len);
