47 /* CONFIG_HID_STATEDEPTH
48  * Constant indicating the maximum stack depth of the state table. A larger
49  * state table allows for more PUSH/POP report items to be nested, but
50  * consumes more memory. By default this is set to 2 levels (allowing non-
51  * nested PUSH items) but this can be overridden by defining
52  * CONFIG_HID_STATEDEPTH in the Nuttx config file.
53  *
54  * CONFIG_HID_USAGEDEPTH
55  * Constant indicating the maximum stack depth of the usage table. A larger
56  * usage table allows for more USAGE items to be indicated sequentially for
57  * REPORT COUNT entries of more than one, but requires more stack space. By
58  * default this is set to 8 levels (allowing for a report item with a count
59  * of 8) but this can be overridden by defining CONFIG_HID_USAGEDEPTH to
60  * in the Nuttx config file.
61  *
62  * CONFIG_HID_MAXCOLLECTIONS
63  * Constant indicating the maximum number of COLLECTION items (nested or
64  * unnested) that can be processed in the report item descriptor. A large
65  * value allows for more COLLECTION items to be processed, but consumes
66  * more memory. By default this is set to 10 collections, but this can be
67  * overridden by defining CONFIG_HID_MAXCOLLECTIONS in the Nuttx config file.
68  *
69  * CONFIG_HID_MAXITEMS
70  * Constant indicating the maximum number of report items (IN, OUT or
71  * FEATURE) that can be processed in the report item descriptor and stored
72  * in the user HID Report Info structure. A large value allows
73  * for more report items to be stored, but consumes more memory. By default
74  * this is set to 20 items, but this can be overridden by defining
75  * CONFIG_HID_MAXITEMS in the Nuttx config file.
76  *
77  * CONFIG_HID_MAXIDS
78  * Constant indicating the maximum number of unique report IDs that can be
79  * processed in the report item descriptor for the report size information
80  * array in the user HID Report Info structure. A large value allows for
81  * more report ID report sizes to be stored, but consumes more memory. By
82  * default this is set to 10 items, but this can be overridden by defining
83  * CONFIG_HID_MAXIDS in the Nuttx config file. Note that IN, OUT and FEATURE
84  * items sharing the same report ID consume only one size item in the array.
85  */
86

272 /****************************************************************************
273  * Name: hid_getitem
274  *
275  * Description:
276  * Extracts the given report item's value out of the given HID report and
277  * places it into the value member of the report item's struct
278  * hid_rptitem_s structure.
279  *
280  * When called on a report with an item that exists in that report, this
281  * copies the report item's Value to it's previous element for easy
282  * checking to see if an item's value has changed before processing a

284  * does not modify the report item's data.
285  *
286  * Input Parameters

299 /****************************************************************************
300  * Name: hid_putitem
301  *
302  * Description:
303  * Retrieves the given report item's value out of the value member of the
304  * report item's struct hid_rptitem_s structure and places it into the
305  * correct position in the HID report buffer. The report buffer is assumed
306  * to have the appropriate bits cleared before calling this function (i.e.,
307  * the buffer should be explicitly cleared before report values are added).
308  *
309  * When called, this copies the report item's Value element to it's
310  * previous element for easy checking to see if an item's value has
311  * changed before sending a report.
312  *
313  * If the device has multiple HID reports, the first byte in the report is
314  * set to the report ID of the given item.
315  *
316  * Input Parameters:
