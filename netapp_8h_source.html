108 /****************************************************************************
109  * Name: netapp_dhcp

111  * Description:
112  * netapp_dhcp is used to configure the network interface, static or
113  * dynamic (DHCP).\n In order to activate DHCP mode, aucIP, aucSubnetMask,
114  * aucDefaultGateway must be 0. The default mode of CC3000 is DHCP mode. Note
115  * that the configuration is saved in non volatile memory and thus preserved
116  * over resets.
117  *
118  * NOTE: If the mode is altered a reset of CC3000 device is required in order
119  * to apply changes.\nAlso note that asynchronous event of DHCP_EVENT, which

121  * due to static allocation is generated only upon a connection to the AP was
122  * established.
123  *
124  * Input Parameters:
125  * aucIP device mac address, 6 bytes. Saved: yes

127  * aucDefaultGateway device mac address, 6 bytes. Saved: yes

138 /****************************************************************************
139  * Name: netapp_timeout_values
140  *
141  * Description:
142  * Set new timeout values. Function set new timeout values for: DHCP lease
143  * timeout, ARP refresh timeout, keepalive event timeout and socket
144  * inactivity timeout
145  *
146  * NOTE: If a parameter set to non zero value which is less than 20s, it will
147  * be set automatically to 20s.
148  *
149  * Input Parameters:
150  * aucDHCP DHCP lease time request, also impact the DHCP renew timeout.
151  * Range: [0-0xffffffff] seconds, 0 or 0xffffffff == infinity
152  * lease timeout.
153  * Resolution: 10 seconds.
154  * Influence: only after reconnecting to the AP
155  * Minimal bound value: MIN_TIMER_VAL_SECONDS - 20 seconds.
156  * The parameter is saved into the CC3000 NVMEM. The default
157  * value on CC3000 is 14400 seconds.
158  * aucARP ARP refresh timeout, if ARP entry is not updated by incoming
159  * packet, the ARP entry will be deleted by the end of the
160  * timeout.
161  * Range: [0-0xffffffff] seconds, 0 == infinity ARP timeout
162  * Resolution: 10 seconds.
163  * Influence: on runtime.
164  * Minimal bound value: MIN_TIMER_VAL_SECONDS - 20 seconds.
165  * The parameter is saved into the CC3000 NVMEM. The default
166  * value on CC3000 is 3600 seconds.
167  * aucKeepalive Keepalive event sent by the end of keepalive timeout
168  * Range: [0-0xffffffff] seconds, 0 == infinity timeout
169  * Resolution: 10 seconds.
170  * Influence: on runtime.
171  * Minimal bound value: MIN_TIMER_VAL_SECONDS - 20 sec
172  * The parameter is saved into the CC3000 NVMEM. The default
173  * value on CC3000 is 10 seconds.
174  * aucInactivity Socket inactivity timeout, socket timeout is refreshed by
175  * incoming or outgoing packet, by the end of the socket
176  * timeout the socket will be closed
177  * Range: [0-0xffffffff] sec, 0 == infinity timeout.
178  * Resolution: 10 seconds. Influence: on runtime.
179  * Minimal bound value: MIN_TIMER_VAL_SECONDS - 20 sec
180  * The parameter is saved into the CC3000 NVMEM. The default
181  * value on CC3000 is 60 seconds.
182  *
183  * Returned Value:
184  * Return on success 0, otherwise error.
185  *
186  ****************************************************************************/
187
